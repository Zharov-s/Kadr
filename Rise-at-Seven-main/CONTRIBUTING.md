# Contributing to Rise At Seven

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/rise-at-seven.git`
3. Install dependencies: `npm install`
4. Create a feature branch: `git checkout -b feat/your-feature`
5. Start dev server: `npm run dev`

## Commit Message Format

Follow conventional commits:
- `feat:` - New feature
- `fix:` - Bug fix
- `docs:` - Documentation
- `style:` - Code style changes
- `refactor:` - Code refactoring
- `perf:` - Performance improvements
- `test:` - Testing
- `chore:` - Maintenance

Example: `feat: add scroll animation to hero section`

## Code Style

- Use ESLint: `npm run lint`
- Follow existing code patterns
- Use meaningful variable names
- Add comments for complex logic

## Pull Request Process

1. Keep PR scope focused on one feature
2. Write descriptive PR title and description
3. Ensure all checks pass
4. Request review from maintainers
5. Address feedback promptly
