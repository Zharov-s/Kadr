# works/ — демо-проекты портфолио freymi

Готовые (собранные) демоверсии работ, на которые ведут карточки в секции
«Портфолио работ» на главной. Чистые URL без `.html` и `/dist/`:

| URL (freymi.ru)      | Папка            | Исходник проекта          | Стек              |
|----------------------|------------------|---------------------------|-------------------|
| `/works/optika/`     | `works/optika`   | `../optic`                | React + Vite      |
| `/works/jiotto/`     | `works/jiotto`   | `../car`                  | Vanilla TS + WebGI (Parcel) |
| `/works/psiholog/`   | `works/psiholog` | `../ENZO-STUDIO-main`     | React + Vite      |

GitHub Pages отдаёт `index.html` автоматически при заходе на директорию,
поэтому в адресной строке видно только аккуратный путь `/works/<slug>/`.
Каждая папка лежит на глубине 2 от корня, поэтому ссылки на главную и общий
подвал работают через `../../` (см. `/shared/freymi-footer.html`).

> Light industrial — внешний сайт (abcport.ru), здесь его нет.

## ⚠️ Пересборка из исходников

Эти папки **доработаны вручную** после сборки: добавлен общий подвал freymi и
фавикон-логотип (см. `/shared/`). Сборщики проектов (`npm run build` / parcel)
по умолчанию пишут в свой `dist/` и регенерируют `index.html` БЕЗ этих правок.

Поэтому при пересборке нельзя просто перезаписать папку `works/<slug>/`. Нужно:
1. Собрать проект из его исходников.
2. Заново внедрить подвал и фавикон из `/shared/` (или вынести их в исходники
   проекта — компонент `<Footer/>` + `<link rel="icon">` в шаблон `index.html`).
3. Положить результат в соответствующую `works/<slug>/`.
